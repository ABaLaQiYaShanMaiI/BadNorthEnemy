### 2.3 错误部分

#### 🔴 错误 #1：Swordsman 不存在 `attackRange` / `idealRange` 字段

**方案中所述**（§7.3）：
```csharp
var arField = typeof(Swordsman).GetField("attackRange", ...);    // ❌ 不存在
var irField = typeof(Swordsman).GetField("idealRange", ...);     // ❌ 不存在
```

**源代码事实**：
```csharp
// Swordsman.cs Line 42-48
public float range
{
    get { return base.agent.radius * 0.7f; }  // ← 计算属性，无 setter，无法修改
}
```

Swordsman 类中的字段：
- `public float[] damageLevels` ✅ — 可用于修改伤害
- `public float[] knockbackLevels` ✅ — 可用于修改击退
- `public float[] stunLevels` ✅ — 可用于修改眩晕
- `[SerializeField] private float attackStaminaCost = 0.5f` ✅ — 可用于修改体力消耗
- `private float stamina = 1f` ✅ — 运行时状态

**父类 CloseCombatBrain**（11行）：
```csharp
public abstract class CloseCombatBrain : Brain
{
    public abstract Attack GetAttack(Agent target);  // 唯一成员
}
```

**正确方案**：要增大攻击距离，只能修改 `agent.radius`：
```csharp
// ✅ 正确：增大 Agent 半径间接扩大攻击范围 + 碰撞体积
agent.radius *= 1.15f;  // Swordsman.range = radius * 0.7f，增大 radius 可增大 range
```

另外可以通过反射修改 `attackStaminaCost` 来改变攻击频率（模拟长矛兵的刺击节奏）：
```csharp
var ascField = typeof(Swordsman).GetField("attackStaminaCost",
    BindingFlags.Instance | BindingFlags.Public | BindingFlags.NonPublic);
if (!ReferenceEquals(ascField, null))
    ascField.SetValue(s, 0.35f);  // 降低单次攻击体力消耗，增加攻击频率
```

## 三、BringBackBerserkers 参考价值评估

### 3.1 BBB 做了什么

```csharp
// BringBackBerserkers/BBB/Plugin.cs
GameSetup.Awake += (orig, self) =>
{
    orig(self);  // ✅ 先执行原逻辑（所有 VikingReference 注册到 dict）
    
    // 从 dict 中取出已存在的 Berserker 预制件引用
    Object obj;
    if (LevelStateObjectReferences.dict.TryGetValue("Viking_Berserker", out obj))
    {
        // 修改 LevelGuessable.probability（从 0 恢复）
        (obj as VikingReference).GetComponent<LevelGuessable>().probability = 
            AxeThrower的probability;
        
        // 修改 LevelRule.condition（从 "false" 恢复）
        (obj as VikingReference).GetComponent<LevelRule>().condition.expression = 
            "(fraction > 0.15 && fraction < 0.4) || ...";
    }
};
```

### 3.2 BBB vs 黑矛兵的根本差异

| 维度 | BBB (BringBackBerserkers) | BlackSpearman |
|------|--------------------------|---------------|
| **目标** | 重新启用已存在的预制件 | 运行时从 SwordShield 转化新兵种 |
| **预制件** | ⭐ 编译器已打包在 AssetBundle 中 | ❌ 没有独立预制件 |
| **VikingAgent.Type** | Beserker 已存在 | 复用 SwordShield（无法新增枚举值） |
| **修改层级** | 数据卡片层（VikingReference） | Agent 实例层（运行时修改组件） |
| **需要武器替换？** | ❌ 不需要（已有独立模型） | ✅ 需要（从 Pikeman 复制） |
| **需要 AI 修改？** | ❌ 不需要（已有独立 Brain） | ✅ 需要（通过 IBrainAction 扩展） |
| **Hook 点** | GameSetup.Awake | Landing.Spawn |

### 3.3 BBB 对黑矛兵的可参考部分

| 可参考 | 不可参考 |
|--------|----------|
| ✅ `GameSetup.Awake` Hook 模式 | ❌ 直接修改预制件字段（黑矛兵无独立预制件） |
| ✅ `LevelStateObjectReferences.dict` 访问模式 | ❌ 恢复禁用的敌人（黑矛兵是新建不是恢复） |
| ✅ `LevelRule.condition.expression` = "true" | ❌ 不需要武器替换 |
| ✅ `LevelGuessable.probability` = "1" | ❌ 不需要 Agent 运行时修改 |
| ✅ `VikingReference` + `LevelRule` + `LevelGuessable` 三件套 | |
| ✅ csproj 引用结构（Assembly-CSharp + BepInEx + MMHOOK） | |

### 3.4 一个 BBB 模式的优点：更简洁的注册方式

BBB 证明了一个关键事实：**不需要创建新的 GameObject + AddComponent，直接修改 dict 中已存在的 VikingReference 就足够了**。

但 BBB 操作的是**已有的、编译器打包的**预制件引用。黑矛兵没有独立预制件，所以 Plugin.cs 中 `RegisterBlackSpearmanReference()` 创建新 VikingReference 的方式是**必要且正确**的。

## 四、技术方案_v1.9 的 3 处源码级错误修正

### 修正 #1：ApplySpearCombatStats() 重写

```csharp
// ❌ 原方案（错误 — attackRange/idealRange 不存在）
private static void ApplySpearCombatStats(Agent agent)
{
    var s = agent.brain as Swordsman;
    SetFloatField(s, "attackRange", 2.5f);   // 字段不存在
    SetFloatField(s, "idealRange", 2.0f);     // 字段不存在
    agent.radius *= 1.1f;
}

// ✅ 修正版
private static void ApplySpearCombatStats(Agent agent)
{
    var s = agent.brain as Swordsman;
    if (ReferenceEquals(s, null)) return;
    
    // 方式1：增大 Agent 半径（间接增大 Swordsman.range = radius * 0.7f）
    agent.radius *= 1.15f;
    
    // 方式2：降低攻击体力消耗（增加攻击频率，模拟长矛兵刺击节奏）
    var ascField = typeof(Swordsman).GetField("attackStaminaCost",
        BindingFlags.Instance | BindingFlags.Public | BindingFlags.NonPublic);
    if (!ReferenceEquals(ascField, null))
    {
        float currentCost = (float)ascField.GetValue(s);
        ascField.SetValue(s, currentCost * 0.7f);  // 降低30%体力消耗
    }
}
```
